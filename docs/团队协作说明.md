# 团队协作说明（AI 开发版）

## 当前状态

- 项目已实现前后端分离。
- 前端与后端可以独立开发、独立调试。
- 生产预览通过后端直接托管前端 `dist`。

## 前端负责（`src/`）

- UI/UX、状态管理、交互流程
- 搜索框输入后的本地关键词过滤
- 详情页缓存优先展示
- 组织 AI 聊天上下文 `conversation_history`

## 后端负责（`server/`）

- 菜单数据读取、分页、排序、筛选
- 推荐链路：意图解析、召回与打分、解释文本
- LLM 调用策略与 fallback 兜底
- API 契约与返回结构稳定性

## 开发优先级（建议）

1. 性能优化（最高优先级）
   - 目标：减少首屏耗时、卡顿和白屏感
2. AI 推荐能力增强
   - 目标：让“推荐”真正成为核心亮点
3. 标签多重筛选
   - 建议放在搜索框下方，支持多条件组合
   - 示例：海棠一楼 + 小炒热菜 + 午餐 + 15~20 元
4. 建议补充改进
   - 增加性能/行为埋点，形成可量化优化闭环
   - 建立推荐评测基线（离线样例 + 人工评分）
   - 完善异常监控（前端错误 + 后端请求链路）
